# Contributing Guidelines

To maintain consistency across the project, please follow these technical standards when contributing 3D models.

## 🛠 Technical Standards

- **Base Models:** We prefer `.blend` (Blender) files for the base model.
- **Textures:** Texture files should be **512x512** pixels.
- **UV Maps:** UV Map templates should be **1024x1024** pixels.
- **Image Formats:** Use `.jpg` or `.png` formats.
- **Source Files:** Please do **not** commit layered `.psd` or `.xcf` files; only include the final exported images.
- **Additional Formats:** Additional model formats (e.g., `.fbx`, `.obj`) are welcome alongside the Blender file.

## ⚖️ Requirements

- **Original Work:** Most importantly, only submit work you have created yourself.
- **Documentation:** Every model folder must include a `[model_name].md` file following the standard template.

## 🔄 Workflow Status Settings

When updating a model's status in its info file, use the following conventions:
- `not started`
- `claimed (user name)`
- `in progress (user name)`
- `complete (user name)`
